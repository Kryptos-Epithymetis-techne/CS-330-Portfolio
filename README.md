# CS-330-Portfolio
CS-330 Computational Graphics and Visualization — 3D scene built with OpenGL and GLFW

How do I approach designing software?

Working on this project has  helped me develop the skills needed for planning out the scene before ever beginning to write the code, allowing me to think through the object placement, lighting, and camera angles as design decisions rather than afterthoughts. I followed an iterative design process, starting simple with basic shapes and gradually layering in textures, lighting, and transformations. This approach of building incrementally and testing it visually at each step is a skill  I can carry into my future software projects, since breaking a large problem into small visible milestones keeps progress manageable.

How do I approach developing programs?

The main new strategy I used was modular development, keeping rendering, input handling, and scene setup in separate functions rather than one large block. Iteration was essential; almost nothing worked perfectly the first time, and each milestone required revisiting earlier code to fix how objects interacted. My approach evolved from writing code linearly to thinking about structure first, which made debugging significantly easier in the later milestones.

How can computer science help me reach my goals?

Computer graphics gave me hands-on experience with how 3D engines work under the hood, like transformations, coordinate systems, or lighting models,  which directly applies to fields like game development, simulation, and data visualization. Professionally, understanding how visual systems are built makes me a stronger candidate in any tech role that involves UI, simulation, or graphics-heavy applications.
